# Data

Maintained by **PRIB-KI Lab**.

## Demo dataset

The bundled demonstration dataset is used to exercise the end-to-end workflow. It is not a validated benchmark set.

Current composition:

- 60 sequences sampled from the public Thera-SAbDab download;
- 40 synthetic stress variants generated from sampled sequences;
- deterministic sampling and mutation with seed `42`;
- neutral candidate identifiers assigned after shuffling.

## Source retrieval

The source download script is `download_data.py`.

Source:

- Thera-SAbDab;
- Oxford Protein Informatics Group;
- therapeutic antibody sequence and structure data.

A future data release should record:

- retrieval date;
- upstream release or snapshot identifier;
- source URL;
- file checksum;
- selected source column;
- filtering summary;
- license or usage conditions.

## Filtering

The current demo builder:

- searches for sequence-like columns;
- requires a minimum sequence length;
- accepts mostly standard amino-acid characters;
- removes duplicate sequences;
- samples the public sequence set with a fixed seed.

## Synthetic variants

Synthetic variants are generated by controlled sequence perturbation intended to shift simple physicochemical descriptors. The current generator can:

- increase local hydrophobicity;
- alter polar patterns;
- reduce or perturb charged residues.

These variants are used to test whether the software reacts to controlled descriptor changes.

They are not:

- experimentally measured failures;
- negative labels;
- evidence of predictive performance;
- suitable for ROC-AUC or accuracy claims.

## Data leakage

Synthetic variants are derived from sampled sequences and are therefore closely related to their parent sequences. They must not be distributed across training and test partitions in a benchmark experiment.

For experimental validation, related sequences should be grouped using sequence clustering, family membership, scaffold, or another biologically meaningful grouping rule.

## TargetTrack historical workflow

TargetTrack is an optional historical source for PRIB's wet-lab failure-funnel
framework, not a replacement for endpoint-specific partner data. Its raw
archive is downloaded locally from its Zenodo DOI, verified against the
published MD5, and excluded from Git. The code, configuration, synthetic
fixtures and aggregate reporting workflow are public; raw and row-level
derived records remain local by default.

The workflow retains only fields required to audit stage and stop
classification. Public outputs must exclude personal contact fields and
unreviewed free text. The official documentation and complete local
raw-status inventory have been inspected, and the software map covers the
observed values. Its biological interpretation remains subject to scientific
review before any training or performance claim. See [the workflow
note](targettrack_wetlab_failure_funnel.md) and [the third-party data
notice](../THIRD_PARTY_DATA.md).

## Intended use

The current dataset supports:

- installation checks;
- UI demonstrations;
- deterministic regression tests;
- feature sensitivity checks;
- internal workflow development.

It must not be presented as evidence that PRIB-KI predicts industrial or clinical protein failure.
